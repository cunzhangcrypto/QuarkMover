# 夸克转存助手 (QuarkMover) - 增强修复版

[![Python Version](https://img.shields.io/badge/python-3.8%2B-blue)](https://www.python.org/)
[![Platform](https://img.shields.io/badge/platform-windows%20%7C%20macos%20%7C%20linux-lightgrey)](https://github.com/)

本项目是在原官方开源版本基础上的深度定制与修复版。针对近期夸克网盘风控升级导致的“扫码无反应”、“扫码请求过期”等问题，进行了登录内核级的重构。

## 🌟 核心改进：为什么选择这一版？

由于夸克网盘近期加强了对无头浏览器（Headless）及自动化特征的检测，原版脚本在登录阶段极易触发风控拦截。本版本做了如下核心优化：

1.  **可见模式登录 (Visible Browser Mode)**：将原有的“静默扫码”改为“弹窗扫码”。通过弹出真实的浏览器窗口，模拟真人操作环境，解决了 99% 的扫码卡死问题。
2.  **深度指纹隐藏**：集成了最新的自动化特征抹除技术（`AutomationControlled` 移除），有效规避浏览器被网站识别为“机器人”。
3.  **双重判定机制**：同时监控 **Cookie (p_token/p_utoken)** 和 **URL 跳转**，只要检测到登录成功，助手将瞬间捕获令牌并自动收网，无需人工干预。
4.  **无痕隔离环境**：每次登录均采用独立的无痕模式，确保多账号添加时互不干扰，不留缓存痕迹。

## 🛠️ 使用说明

### 1. 账号添加流程
* 点击软件内的 **“添加账号”**，系统会弹出一个 Chrome 浏览器窗口。
* 请使用夸克 App 扫描窗口中的二维码并点击确认。
* **注意**：扫码成功后，浏览器会自动进入网盘主页，此时请**静候 2-3 秒**。助手识别成功后会自动关闭该窗口。

### 2. 关于“不受支持”的提示
启动浏览器时，顶部若出现 `“您使用的是不受支持的命令行标记...”`：
* **请忽略它！** 这是防封号伪装参数生效的正常表现，不影响正常扫码和安全性。

### 3. 多账号管理
* 助手完美支持多账号登录。添加完一个账号并显示成功后，可继续点击添加下一个。
* 所有的转存、移动任务依然维持原作者的高效静默逻辑，不会产生额外弹窗。

## 📋 环境要求
* 已安装 **Google Chrome 浏览器**（建议保持最新版本）。
* Python 3.8 或更高版本。
* 依赖库：`DrissionPage`, `httpx`, `loguru` 等（详见原版 `requirements.txt`）。

## ⚠️ 注意事项
* **请勿频繁登录**：Cookie 抓取成功后会自动保存，除非账号失效，否则无需反复扫码。
* **路径设置**：请确保 Chrome 已安装在默认路径，或在配置文件中正确指向您的浏览器可执行文件。

## 常见问题

**Q: 提示"未找到浏览器"？**
A: 请安装 [Google Chrome](https://www.google.com/chrome/) 或使用系统自带的 Edge 浏览器。

**Q: 转存失败，提示"未登录"？**
A: 登录态已过期，点击右上角「扫码登录夸克」重新登录。

**Q: 二创功能报错？**
A: 需要在设置中填写有效的 DeepSeek API Key（[获取地址](https://platform.deepseek.com/)）。
